# AsteroidsFX — Folder Structure

```
AsteroidsFX/
│
├── pom.xml                          # Root POM — declares all modules, manages dependencies & exec plugin
├── nbactions.xml                    # NetBeans run/build action bindings
├── README.md
├── CLAUDE.md
│
├── Common/                          # Shared library — no game logic
│   ├── pom.xml
│   └── src/main/java/
│       ├── module-info.java         # exports dk.sdu.mmmi.cbse.common.*
│       └── dk/sdu/mmmi/cbse/common/
│           ├── data/
│           │   ├── Entity.java      # Base class for all game objects (position, rotation, polygon, radius)
│           │   ├── GameData.java    # Shared state: display dimensions, GameKeys reference
│           │   ├── GameKeys.java    # Keyboard state tracker (LEFT, RIGHT, UP, SPACE)
│           │   └── World.java       # ConcurrentHashMap entity registry (add/remove/getEntities)
│           ├── services/
│           │   ├── IGamePluginService.java          # start()/stop() lifecycle interface
│           │   ├── IEntityProcessingService.java    # process(GameData, World) per-frame interface
│           │   └── IPostEntityProcessingService.java# Same signature; runs after all entity processors
│           └── util/
│               └── ServiceLocator.java              # Static helper for ServiceLoader lookups
│
├── CommonAsteroids/                 # Shared asteroid types — consumed by Asteroids & Collision
│   ├── pom.xml
│   └── src/main/java/
│       ├── module-info.java
│       └── dk/sdu/mmmi/cbse/common/asteroids/
│           ├── Asteroid.java        # Entity subclass representing an asteroid
│           └── IAsteroidSplitter.java # SPI: splits an asteroid into smaller ones on collision
│
├── CommonBullet/                    # Shared bullet types — consumed by Player, Bullet & Collision
│   ├── pom.xml
│   └── src/main/java/
│       ├── module-info.java
│       └── dk/sdu/mmmi/cbse/common/bullet/
│           ├── Bullet.java          # Entity subclass representing a bullet
│           └── BulletSPI.java       # SPI: createBullet(Entity shooter, GameData) — implemented by Bullet module
│
├── Core/                            # Application entry point and game loop
│   ├── pom.xml
│   └── src/main/java/
│       ├── module-info.java         # uses all three service interfaces
│       └── dk/sdu/mmmi/cbse/main/
│           ├── Main.java            # JavaFX Application subclass; bootstraps Spring context
│           ├── ModuleConfig.java    # @Configuration; loads plugins via ServiceLoader into Spring beans
│           └── Game.java            # @Component; owns AnimationTimer, calls processors each frame, manages Polygons
│
├── Player/                          # Player ship plugin
│   ├── pom.xml
│   └── src/main/java/
│       ├── module-info.java         # provides IGamePluginService, IEntityProcessingService; uses BulletSPI
│       └── dk/sdu/mmmi/cbse/playersystem/
│           ├── Player.java          # Entity subclass for the player ship
│           ├── PlayerPlugin.java    # IGamePluginService: creates Player entity on start, removes on stop
│           └── PlayerControlSystem.java # IEntityProcessingService: handles arrow key movement & SPACE to fire
│
├── Asteroids/                       # Asteroid plugin
│   ├── pom.xml
│   └── src/main/java/
│       ├── module-info.java         # provides IGamePluginService, IEntityProcessingService, IAsteroidSplitter
│       └── dk/sdu/mmmi/cbse/asteroid/
│           ├── AsteroidPlugin.java       # IGamePluginService: spawns initial asteroid on start
│           ├── AsteroidProcessor.java    # IEntityProcessingService: moves asteroids, wraps at screen edges
│           └── AsteroidSplitterImpl.java # IAsteroidSplitter: splits hit asteroid into two smaller ones
│
├── Bullet/                          # Bullet plugin
│   ├── pom.xml
│   └── src/main/java/
│       ├── module-info.java         # provides IGamePluginService, IEntityProcessingService, BulletSPI
│       └── dk/sdu/mmmi/cbse/bulletsystem/
│           ├── BulletPlugin.java        # IGamePluginService: lifecycle stub
│           └── BulletControlSystem.java # IEntityProcessingService + BulletSPI: moves bullets, creates them on request
│
└── Collision/                       # Collision detection post-processor
    ├── pom.xml
    └── src/main/java/
        ├── module-info.java          # provides IPostEntityProcessingService
        └── dk/sdu/mmmi/cbse/collisionsystem/
            └── CollisionDetector.java # Checks all entity pairs by radius overlap; removes colliding entities,
                                       # loads IAsteroidSplitter via ServiceLoader to split asteroids on bullet hit
```

## Runtime output directories (generated by Maven, not in source control)

```
AsteroidsFX/
├── mods-mvn/    # Each module's JAR is copied here by the child-project Maven profile
└── libs/        # External dependency JARs (JavaFX, Spring) copied here at build time
```

Both directories are placed on the Java module path / class path by the exec-maven-plugin when running `mvn exec:exec`.
